## 落地原则

1. 结果页需要同时具备“可分享性”和“可训练性”。
2. 评分规则必须可解释（避免黑盒）。
3. 全段位适配：避免过度偏向新手或高分技巧词汇。
4. 推荐逻辑不固定偏保守/激进，而是根据风险偏好与团队倾向动态输出。
5. 交付以静态站点优先，支持直接部署到 GitHub Pages。
